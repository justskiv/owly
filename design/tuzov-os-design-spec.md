# TuzovOS — Дизайн-спецификация (финал)

## Цветовая система

### Философия
Нейтральный серый холст (R=G=B, без цветового оттенка) + живые цветовые акценты. Вся «краска» — только в категориях и золотом акценте.

### Поверхности (чистый серый R=G=B)
| Роль | Hex | Где |
|------|-----|-----|
| Deep | `#131313` | Sidebar, пул задач, statusbar |
| Base | `#1A1A1A` | Сетка планировщика, хедер |
| Surface | `#222222` | Detail-панель сущностей, модалки |
| Elevated | `#2B2B2B` | Карточки списка, dropdown |
| Hover | `#353535` | Hover-состояния |
| Active | `#3F3F3F` | Нажатие |

### Текст — 5 уровней
Вся текстовая шкала сведена к 5 токенам. Никаких inline `#fff`/`#e0e0e0`/`#c8c8c8`/`#c0c0c0`/`#999`/`#888` в CSS — только через `var(--text-*)`.

| Токен | Hex | Где |
|-------|-----|-----|
| `--text-primary` | `#F2F2F2` | Заголовки detail-карточек, метрики, числа стат-карточек, названия блоков, body-текст, описания, history-текст контактов |
| `--text-secondary` | `#B0B0B0` | Мета-информация, section-titles (UPPERCASE), note-body-текст, ct-item |
| `--text-tertiary` | `#808080` | Labels полей detail, подписи стат-карточек, goal-sub, часы на временной шкале, зачёркнутые done-пункты чеклистов (AA-контраст + strikethrough) |
| `--text-disabled` | `#585858` | Плейсхолдеры, метаинформация шкал/футеров (не для интерактивного текста) |
| `--text-inverse` | `#131313` | Текст на золотом акценте (btn-save, today-кружок) |

### Акцент — `#E0B860` (тёплое золото)
Используется только для сигналов внимания: sidebar active indicator, today-кружок в day header, now-линия, выделение выбранного блока/строки/фильтра, pipeline current-стадия, streak today-рамка, primary-кнопки («Сохранить», «+ Создать», «Редактировать»).

Декоративные и done-состояния **намеренно не используют акцент**: буллеты и чекбоксы заметок, дедлайн-бейдж в нейтральном, пройденные стадии pipeline, число прогресса цели — идут через текстовые токены и `--success`. Золото должно оставаться «дорогим» сигналом: где золото — туда смотрят.

### Семантика
Success `#30D888` · Error `#E06878`

Warning-токен удалён намеренно: две его роли разведены семантически. Overlap-блок (конфликт расписания) классифицирован как error (красный dashed border). Priority MEDIUM использует нейтральный `--text-secondary` — получается ровная шкала важности red → grey → dark-grey.

### Белые opacity — 5 уровней
Вся шкала прозрачных белых тонов сведена к пяти токенам — два для поверхностей/фонов, три для границ. Никаких inline `rgba(255,255,255, X)` в коде (кроме определений в `:root`).

| Токен | Значение | Применение |
|-------|----------|------------|
| `--bg-tint-1` | `rgba(255,255,255, 0.03)` | Еле видимый фон: hover пустых ячеек, subtle surface, стандартные bg чипов |
| `--bg-tint-2` | `rgba(255,255,255, 0.05)` | Emphasized hover, stat-card, ct-status, pi-hover, snap-preview fill, pipe-bar track |
| `--border` | `rgba(255,255,255, 0.08)` | Линии сетки, разделители секций, scrollbar-thumb |
| `--border-default` | `rgba(255,255,255, 0.14)` | Инпуты, бордеры карточек, scrollbar-thumb:hover, drag-ghost border |
| `--border-strong` | `rgba(255,255,255, 0.22)` | Hover на инпутах, resize-handle :hover, snap-preview dashed, ctx-c.active border |

### Размеры шрифтов — 7 токенов
Свёрнуто с 14 разрозненных значений (включая дробные 9.5/10.5/11.5) до 7 чётких уровней.

| Токен | Значение | Применение |
|-------|----------|------------|
| `--fs-2xs` | `10px` | Labels uppercase, mono-tiny: time-lbl, hm-month/days/legend, sbar, dh-name, wstats, bm, bn |
| `--fs-xs` | `11px` | Small text, бейджи, pool items, filters, mono secondary: er-badge, edp-tag, edp-deadline, ctx-sc, hdr-btn-ghost |
| `--fs-sm` | `12px` | Default small: ctx-i, fi labels, hdr-btn, goal-sub, pi-t, m-title |
| `--fs-md` | `14px` | Body, titles: body default, hdr-week, hdr-title, er-title, ct-status-title, edp-desc, note-body, search-input, edp-val, edp-cl li, stat-f-val |
| `--fs-lg` | `16px` | goal-target |
| `--fs-xl` | `20px` | edp-title, n-h1 в заметках, stat-num (было 22) |
| `--fs-2xl` | `28px` | goal-cur, крупные числа |

**Inline-исключения** (1 место каждое, отдельного токена не нужен):
- `36px` на `.metric-val` — крупное числовое значение метрики
- `24px` на `.ct-cd-num` — countdown-число в статус-виджете контакта
- `8px` на `.note-body .n-bullet` — глиф-буллет `●` (декоративный, не текст)

Круги (`border-radius: 50%`) оставлены inline — это не «скруглённые прямоугольники», а декоративная геометрия.

### Spacing — 4px-сетка
Все `padding`, `gap`, `margin` в CSS строго на шагах **4 / 8 / 12 / 16 / 24 / 32 px**. Магические значения `5 / 6 / 7 / 9 / 10 / 14 / 18 px` удалены при рефакторинге B5.

| Значение | Применение |
|----------|------------|
| `4px` | tight gap/padding (бейджи, list items, чекбоксы, pool items) |
| `8px` | стандартные gap/padding (кнопки, чипы, inline-контролы, меню) |
| `12px` | комфортные row paddings (erow, pool-hd, ct-week-item, header gap) |
| `16px` | секции/карточки (edp-sec вертикально, modal, stat-card, note-body, status-widget) |
| `24px` | edp-sec горизонтально, edp-actions, edp-head |
| `32px` | зарезервирован для больших блоков (spark-x padding-left) |

**Inline-исключения** (не магия — точки между токенами):
- `1px / 2px / 3px` — декоративные: hairlines, bullet-offsets, micro-margins (wsbar height, .bm padding-left, .tb left/right offset)
- `20px` — `.fs margin-bottom`, `.dash-grid padding`, `.s-logo margin-bottom` (хвосты filter-секций и дашбордов)
- `28px` — `.s-logo 28×28`, heatmap `margin-left:28px` (label gutter)
- `40px` — `.edp-empty padding` (центрирование заглушки)
- `48px` — `.hdr height` (стандартная высота header-бара)

Значения 20/28/40/48 остаются на 4-grid, но между токенами — «между-размеры» для специфичных кейсов, не используются повсеместно.

## Доступность

### Контрастные требования (WCAG 2.1)
| Тип контента | AA минимум | AAA целевой |
|--------------|-----------|-------------|
| Body text (≥14px regular, ≥18px bold) | 4.5:1 | 7:1 |
| Large text (≥18px regular, ≥14px bold) | 3:1 | 4.5:1 |
| UI labels, captions | 4.5:1 | — |
| Decorative, disabled | не регулируется | — |

### Контраст текстовых токенов на `--bg-base` (`#1A1A1A`)
| Токен | Цвет | Контраст | Уровень |
|-------|------|----------|---------|
| `--text-primary` | `#F2F2F2` | 14.8 : 1 | AAA ✓ |
| `--text-secondary` | `#B0B0B0` | 8.3 : 1 | AAA ✓ |
| `--text-tertiary` | `#808080` | 4.6 : 1 | AA (на грани) |
| `--text-disabled` | `#585858` | 2.3 : 1 | только для disabled-состояний |

### Фокус
- Токен: `--focus-ring: var(--accent)` — золотой outline как attention-сигнал (переиспользует семантику accent)
- Правило: `button:focus-visible, [tabindex]:focus-visible { outline: 2px solid var(--focus-ring); outline-offset: 2px; border-radius: inherit }`
- Фокус виден **только при keyboard-навигации** (не при клике мышью) благодаря `:focus-visible`
- Все интерактивные элементы фокусируются по Tab:
  - Нативные: `button`, `input`, `a`
  - Кастомные div-кнопки через `tabindex=0`: `.si` (sidebar), `.tb` (блоки), `.pi` (пул), `.fo` (фильтры), `.erow` (строки сущностей)
- Enter/Space на focused div-кнопке триггерит тот же callback, что click (реализовано через хелпер `makeClickable`)
- Inputs сохраняют свой `:focus { border-color }` change — outline для них не добавляем (избыточно)

## Экран 1 — Планировщик

### Недельная сетка
- 7 дней × 17 часов (06:00–23:00)
- Row height: **40px** (CSS `--row-h` и JS `R` синхронизированы)
- Snap: **30 минут**, минимальная длительность блока: 30 мин

### Day headers
- Название дня (Пн, Вт...) + число
- **Balance bar** в каждом day header: фоновый трек `var(--bg-tint-2)`, цветные секции слитные (overflow:hidden, без gap), часы справа
- Today: число в **золотом кружке** (`--accent`)

### Week summary bar
- В хедере между навигацией и кнопками
- Stacked bar (те же цвета категорий) + справа часы по категориям

### Now-линия
- Цвет: **`var(--accent)`** (золотая), 2px толщина
- Позиция задана константой `NOW_MIN` (демо — без `setInterval`)

### Блоки расписания
- Заливка 35% цвета категории → hover 45% (health на 3% меньше — зелёный канал ярче на чёрном фоне). Active-now блок: 55% / 62% hover
- Title: 12px Outfit 500, time: 10px JetBrains Mono 300
- Без левой полоски

### Drag & Drop
- Mousedown: если < 5px движения → клик (select), иначе drag
- Ghost сохраняет **offset от точки захвата** (grabOffX/grabOffY)
- Drop target detection: `cy - r.top` **без двойного scrollTop** (getBoundingClientRect уже учитывает)

### Resize
- Зона: нижние **8px** блока, курсор `ns-resize`
- Tooltip длительности у курсора при drag
- Snap 30 мин, минимум 30 мин

### Drag из пула
- Ghost следует за курсором
- Drop создаёт блок, задача исчезает из пула

### Inline-создание
- Клик на пустую ячейку → инлайн-блок с полем ввода
- Enter → создать (категория work по умолчанию), Escape → отмена
- **Hover на пустой ячейке:** 2px вертикальная полоска `--work` слева (`opacity .4`) — affordance дефолт-категории создаваемого блока

### Overlap detection
- `border: 1px dashed var(--error)` + ⚠ в правом верхнем углу

### Контекстное меню (ПКМ на блоке)
```
✓ Выполнено         D
✗ Пропущено         S
──────────────────────
✎ Редактировать     Enter
⧉ Дублировать       ⌘D
✕ Удалить           ⌫
──────────────────────
Категория →  ● ● ● ● ●  (цветные кружки)
```

### BlockEditor (модалка)
- Ширина 400px, backdrop-blur
- Поля: Название, Начало, Длительность (мин), День (0–6), Область (радио с цветами категорий), Заметки
- Кнопка «Сохранить» — **золотая** (`--accent`), «Удалить» — красная outline
- Enter = сохранить, Escape = закрыть

### Пул задач (264px)
- Сворачивается кнопкой или `T`
- Группировка по приоритету (Высокий/Средний/Низкий)
- Элементы строятся через **createElement + appendChild** (не innerHTML +=, чтобы не убивать listeners)
- Draggable на сетку

## Экран 2 — Сущности (Master-Detail)

### Layout
Фильтры (180px) | Список (380px) | Detail-панель (flex:1, max-width 480px на секции)

### Фильтры (sidebar)
- **По типу** (8 пунктов): Все, 📋 Задачи, 📁 Проекты, 🔄 Рутины, 👤 Контакты, 🎯 Цели, 📈 Метрики, 📝 Заметки (отдельной вкладки 📅 События нет — в data 1 событие, доступно через «Все»)
- **По области**: Work, People, Life, Growth, Health (цветные точки)
- **По статусу**: Active, Someday, Done
- Active filter: золотой акцент фона

### Список — компактные строки
- Иконка + title + badge приоритета (HIGH красный pill / MEDIUM жёлтый / LOW серый)
- Под title: цветные теги + мета-инфо
- Кнопки: **«+ Создать»** с золотым акцентом

### Detail-панель — общее

**Шапка:** заголовок (`--fs-xl` `var(--text-primary)`) → цветные теги (фон category+22%, скруглённые pills)

**Бейджи в строке:**
- Тип с иконкой (📁 Проект / 📋 Задача / ...)
- Статус (Active зелёный / Someday серый)
- Приоритет (HIGH красный pill / MEDIUM жёлтый / LOW серый)
- Дедлайн (нейтральный серый pill с 📅, красный если просрочен)

**Кнопки:** «Редактировать» (золотая primary) + «Удалить» (красная outline)

### Detail — виджеты по типам

**Проект:**
- Pipeline: все стадии с подписями (текущая — золотая, пройдённые — приглушённые серые, будущие — почти прозрачные), крупное название текущей стадии сверху
- Чеклист глав с прогресс-баром
- Заметки

**Задача:**
- Бейджи (статус, приоритет, дедлайн)
- Чеклист подзадач с прогрессом
- Заметки

**Рутина:**
- Stat-карточки: стрик, % выполнения, частота
- **Streak-неделя:** 7 квадратиков Пн–Вс (done = зелёный `--success`, missed = серый, today = золотая рамка `--accent`)
- **GitHub-style heatmap:** 26 недель, месяцы сверху, дни (Пн/Ср/Пт) слева, 4-уровневая легенда «Меньше → Больше»

**Контакт:**
- **Статус-виджет:** кольцо ✓/!, обратный отсчёт («Через N дн.» / «Просрочено N дн.»), крупная цифра
- **«На этой неделе»:** запланированные блоки с заливкой цвета категории
- **Чеклист:** чекбоксы + hover-dismiss (×), toggle чекбоксов
- **История контактов:** таймлайн с иконками и датами
- **Важные даты:** 🎂/💐/🤝 — **без фонового бокса**, только hover-подсветка

**Цель:**
- **Прогресс-блок:** 33K крупно слева, 55K справа, progress bar 8px (inline override; дефолт `.edp-pbar` — 6px, используется в чеклистах), подпись «60% · осталось 22K · дедлайн 31 дек 2026»
- **Футер:** три колонки — тип цели / темп (зелёным) / прогноз
- **Связанные метрики:** кликабельные строки, **без фона** (transparent), hover → фон `var(--bg-tint-2)`, стрелка → **только при hover** (opacity:0 → 1)
- **Sparkline:** оси Y (значения) и X (месяцы), grid-линии, точки данных

**Метрика:**
- Крупное значение (36px mono `var(--text-primary)`)
- Индикатор изменения (↑/↓ зелёным — направление зависит от метрики)
- **Столбчатый тренд 6 мес.:** pixel heights (chartH=90px, baseline = minV - spread×0.25), gradient opacity (от 0.35 до 1.0), контейнер 120px
- Футер: единица / ср. рост / связанная цель
- 3 примера: YouTube подписчики (28→33K), Доход ($620→$1,240), Вес (78→74.2кг)

**Заметка:**
- Свободное форматирование в **едином контейнере** (bg `rgba(255,255,255,.015)` — намеренно тише чем `--bg-tint-1`, еле заметная тень под текстом; border `--bg-tint-2`, border-radius `--radius-lg`, padding 16px симметричный)
- Элементы: H1 (20px bold = `--fs-xl`), H2 (14px bold = `--fs-md`), буллеты (● в цвете первой категории заметки через CSS-переменную `--note-accent` + `opacity: .55` — тонированный намёк на категорию, без агрессии), чекбокс unchecked (обводка `--text-tertiary`), чекбокс checked (обводка + fill + галочка в том же `--note-accent`, что и буллеты — симметрия тональности внутри заметки, Raycast-like), bold, italic, hr-разделители
- H1/H2 без top-margin когда первый элемент
- Футер: «обновлено дата» + «N слов · M символов» — **без разделительной линии** между телом и футером

| Клавиша | Действие |
|---------|----------|
| `N` | Новый блок (модалка) |
| `T` | Toggle пула |
| `D` | Блок → Done |
| `S` | Блок → Skipped |
| `Delete` | Удалить блок |
| `Enter` | Редактировать блок |
| `Ctrl+↑/↓` | Сдвинуть блок на 30 мин |
