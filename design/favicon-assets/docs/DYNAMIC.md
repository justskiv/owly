# Dynamic date: как это работает и где границы

## Web (браузерный favicon)

### Механика
1. `dynamic-favicon.js` держит SVG-шаблон в строке с плейсхолдером `__DAY__`.
2. На старте берёт `new Date().getDate()`, подставляет, кодирует в `data:` URL,
   создаёт/обновляет `<link rel="icon" type="image/svg+xml">`.
3. `setTimeout` на следующую локальную полночь (+5 сек буфер) → повтор.
4. `visibilitychange` → если вкладка вернулась из фона, форсируем перерисовку
   (на случай, когда комп спал всю ночь и `setTimeout` не сработал вовремя).

### Caveats
- **Кеш.** Chrome и Safari агрессивно кешируют favicon. `data:` URL с новым
  содержимым обходит кеш: это новый URL → новый ресурс. Работает надёжно.
- **Шрифт.** В SVG используется `font-family` со стеком `-apple-system, Helvetica
  Neue, Helvetica, Arial, sans-serif`. Рендерится браузером на лету — на macOS
  это SF Pro, на Windows Segoe UI, на Linux DejaVu. Лёгкие расхождения в рисунке
  цифры есть, но композиция и контраст одинаковые.
- **Size.** Браузер скейлит SVG под нужный размер таба (обычно 16×16, у
  pinned-табов 32×32). Векторный рендер адекватен.
- **Dark mode браузера.** Тёмная карточка #131313 с rx=7 смотрится как нативная
  иконка и на светлом, и на тёмном табе. Светлый таб видит dark card, тёмный
  таб — почти сливается (что нормально — контраст держит золотая шапка и цифра).

## Tauri (macOS Dock)

### Что где живёт
- `.icns` в `.app/Contents/Resources/` — **статичный**. Это иконка, которую
  видно в Finder, Launchpad, Spotlight и в Dock-слоте **закрытого** приложения.
  Пересобирается только с новой сборкой приложения.
- **Dock-tile живого процесса** — меняется через `NSApp.setApplicationIconImage(_:)`.
  Действует только пока процесс жив. При следующем запуске Dock сначала покажет
  bundle-иконку, пока код не выставит новую.

### Механика динамического обновления
1. `dynamic_icon::spawn_midnight_updater()` в `setup(|_app|)` стартует фоновый поток.
2. Поток:
   - берёт `Local::now().day()`
   - рендерит SVG-шаблон через `resvg` в PNG 1024×1024 в памяти
   - конвертирует в `NSImage`, вызывает `[NSApp setApplicationIconImage:]`
   - спит до следующей локальной полуночи +5 сек
   - повторяет
3. SVG-шаблон вкомпилирован через `include_str!` — отдельного файла не нужно
   на устройстве пользователя.

### Caveats
- **До первого запуска** — Dock показывает bundle-иконку (дата в момент последней
  сборки). Т.е. если юзер обновил приложение 15-го, и не запускал до 20-го, то
  в Dock будет 15, пока не запустит.
- **Quit-состояние** — Dock-слот закрытого приложения показывает bundle-иконку,
  не последнюю динамическую. Это ограничение macOS, обойти нельзя без фонового
  Launch Agent (не стоит усложнять ради иконки).
- **Sleep через полночь** — если Mac спал в момент `sleep` внутри потока, таймер
  промахнётся. Решение: при `NSWorkspaceDidWakeNotification` форсировать апдейт.
  Это +15 строк в `dynamic_icon.rs` через NotificationCenter subscribe. Если
  нужно — добавлю, но на практике `spawn_midnight_updater` просто обнаружит
  пропущенную полночь при следующем проходе цикла (sleep вернёт
  управление позже чем рассчитывалось, следующая итерация прочитает уже новую
  дату).
- **Шрифт в `resvg`** — подгружается через `fontdb.load_system_fonts()`. На macOS
  найдёт Helvetica Neue по стеку — совпадает с тем, что рендерит Safari, т.е.
  Dock-иконка визуально идентична favicon-у.

### Почему не тронуть `.icns` в бандле
Технически можно пересобирать `.icns` каждую полночь и заливать в
`Contents/Resources/AppIcon.icns`, но:
- это требует write-доступа к `.app` (на подписанной и нотарифицированной сборке
  — поломает подпись)
- macOS агрессивно кешит bundle icons (`/var/folders/.../com.apple.iconservices.store`)
- результат непредсказуем

Выставление live Dock-tile через `NSApp` — штатный API, без ломки подписи и кеша.

## Альтернатива: badge вместо полной смены

Если полная смена иконки кажется избыточной, можно оставить **статичную**
`.icns` (скажем, с «15» или без цифры вовсе) и дорисовывать **сегодняшнее число**
как бейдж через `[dockTile setBadgeLabel:]`:

```rust
let label: id = NSString::alloc(nil).init_str(&today.to_string());
let _: () = msg_send![NSApp().dockTile, setBadgeLabel: label];
```

Плюсы: дешевле, не надо рендерить PNG каждую полночь.
Минусы: нативный macOS бейдж — красный, не кастомизируется. Может смотреться
как notification-счётчик, не как дата.

## Рекомендация

- **Web:** `dynamic-favicon.js` как есть.
- **Tauri macOS:** `spawn_midnight_updater` как есть + (опционально)
  `NSWorkspaceDidWakeNotification` subscriber, если заметишь, что после сна
  иконка отстаёт.
- **Windows/Linux:** поведение аналогичное через соответствующие API
  (`SetConsoleIcon`/`SHChangeNotify` на Win, `_NET_WM_ICON` на X11) — но до
  ситуации, когда это реально нужно, отложить.
