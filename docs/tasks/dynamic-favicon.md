# Динамическая иконка приложения (Tauri)

> **Статус:** отложено. В стартовых фазах используется статический
> favicon из `design/favicon-assets/tauri/icons/`. Динамическая
> иконка — бонус-фича, если понадобится.

## Что это

В `design/favicon-assets/` дизайнер подготовил не только статические
PNG/ICNS, но и Rust-модуль `tauri-rust/dynamic_icon.rs` + интеграцию
в `main.rs`, который позволяет менять иконку в доке macOS в
реальном времени.

Варианты использования:
- Иконка меняется в зависимости от текущего блока в расписании
  (цвет категории)
- Индикация состояния: «идёт AI-планирование», «есть ошибка в
  очереди команд»
- Показ прогресса или числа непрочитанных задач

## Что уже готово

В `design/favicon-assets/`:
- `tauri-rust/dynamic_icon.rs` — модуль рендеринга иконки в
  runtime (использует `resvg-js`)
- `tauri-rust/main-integration.rs` — пример интеграции в `main.rs`
- `tauri-rust/Cargo-additions.toml` — дополнительные зависимости
- `web/dynamic-favicon.js` — js-helper для вызова из фронта
- `docs/DYNAMIC.md` — документация

## Когда делать

Не раньше фазы 6 (AI Integration). Сначала нужно, чтобы приложение
было полностью рабочим, и только потом добавлять косметические
улучшения.

Альтернативно — вообще вынести в отдельный этап «Polish» после
всех фаз, вместе с другими мелкими улучшениями (анимации,
звуковые нотификации, и т.п.).

## План (когда дойдут руки)

1. Прочитать `design/favicon-assets/docs/DYNAMIC.md`
2. Добавить в `src-tauri/Cargo.toml` зависимости из
   `Cargo-additions.toml`
3. Перенести `dynamic_icon.rs` в `src-tauri/src/`
4. Интегрировать в `main.rs` по образцу `main-integration.rs`
5. Добавить Tauri-команду `update_icon(color, text)` для вызова
   из фронта
6. Из JS/TS вызывать через `invoke('update_icon', ...)` в
   моменты:
   - Смена текущего блока (каждую минуту или по событию)
   - Старт/финиш выполнения AI-команды
   - Переход в режим ошибки (красный оттенок)

## Чем можно ограничиться для первой версии

Если полная динамика кажется перебором — достаточно **двух**
состояний:
1. Базовая иконка (как сейчас)
2. Иконка с красным индикатором при наличии failed-команд

Это закрывает основной кейс нотификации пользователя без запуска
resvg-рендерера в runtime.
