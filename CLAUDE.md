# TuzovOS

Десктопное приложение на Tauri v2 + React + TypeScript. Персональный
центр управления: недельный планировщик, сущности с тегами, дашборды.

Спецификация — в `spec/tuzov-os/`.

## Workflow по фазам

Перед началом любой фазы:

1. Прочитай `spec/tuzov-os/00-overview.md` — общая картина
2. Прочитай `spec/tuzov-os/01-data-schema.md` — схемы данных
3. Прочитай `spec/tuzov-os/02-architecture.md` — архитектура
4. Прочитай спеку конкретной фазы (`03-…` … `08-…`)
5. Составь план реализации и покажи **до** старта работы

После завершения фазы:

- Пройдись по чеклисту в конце спеки фазы
- Убедись, что `npm run tauri dev` запускается без ошибок
- Сообщи, что готово, дождись пока заберу

## Принципы реализации

- Строго следуй спецификации текущей фазы. Не добавляй фич, которых
  в спеке нет
- Не делай заглушек: если спека требует компонент — реализуй полностью
- Все данные — JSON-файлы в `data/`. Никаких баз данных
- Zod-схемы — источник правды для типов: сначала схема, потом код
- Валидация через Zod при каждом чтении файла
- Запись в файлы атомарная (temp + rename)
- Тёмная тема по умолчанию
- Tailwind для стилей, Zustand для стейта

## Стиль кода

- TypeScript strict mode
- Функциональные компоненты с хуками
- Имена файлов: `PascalCase` для компонентов, `camelCase` для утилит
- Код и идентификаторы — на английском; комментарии на русском
  допустимы
- Без лишних абстракций. Простой код лучше «правильного»

Подробные принципы и анти-паттерны — в `CODESTYLE.md`. Каждое правило
оттуда связано с реальным кейсом ревью. В обсуждении PR — ссылаться
номером пункта.
