# TuzovOS — Обзор проекта

## Что это

Десктопное приложение — персональный центр управления жизнью и работой. Не таск-менеджер, не календарь, не CRM. Это единая система, где пересекаются все сферы жизни: работа, люди, быт, развитие, здоровье — и где из этого хаоса собирается конкретный план на неделю.

Ключевая идея: **данные открыты** (JSON + Markdown на диске), **приложение — UI поверх них**, **AI-агент работает снаружи** (Claude Code / Cowork) и может читать данные напрямую, а писать — через очередь команд.

## Для кого

Для одного пользователя. Но архитектура универсальная — без привязки к конкретному человеку. Все персональные данные в `data/`, движок не знает о них ничего. Потенциально open-source.

## Три слоя приложения

### 1. Недельный планировщик (ядро, ежедневное использование)
- Сетка 7 дней × 24 часа, шаг 30 минут
- Блоки произвольной длительности, drag-and-drop
- Пул задач сбоку — всё незапланированное из всех сфер
- Баланс дня: сколько часов на что потрачено
- Шаблон рутин: собаки, завтрак, японский — заполняются автоматически
- Три режима: ручной, AI-планирование, AI-ревью

### 2. Данные по областям (контекст)
- Сущности типизированы: задача, проект, рутина, событие, контакт, цель, заметка, метрика
- Тегированы по областям: work, people, life, growth, health (+ пользовательские)
- Одна сущность может иметь несколько тегов — нет проблемы "куда положить"
- Области = не папки, а фильтры/виды

### 3. Дашборды (витрины)
- React-компоненты, которые генерирует AI-агент
- Приложение просто рендерит готовые `.jsx` файлы из папки `dashboards/`
- Добавляются по мере надобности, как гугл-таблицы
- Стартовый набор: KPI, планы по месяцам, пайплайн контента

## Принципы

1. **Файлы — источник истины.** JSON + Markdown на диске. Никаких баз данных. Git для версионирования.
2. **Приложение — тупой UI.** Читает файлы, рендерит, пишет обратно. Детерминированная логика (overlap detection, time math, validation) живёт в приложении. Всё «умное» — в агенте.
3. **AI снаружи.** Никакого встроенного чата, стриминга, управления контекстом. Агент (Claude Code) работает с файлами. Приложение не знает о существовании AI.
4. **Тегирование вместо папок.** Двухосевая модель: тип сущности × область жизни. Сущность принадлежит области через теги, а не через расположение в файловой системе.
5. **Команды для записи.** Агент читает файлы напрямую, но пишет через очередь команд (JSON-файлы в `commands/pending/`). Приложение подхватывает, валидирует, исполняет.
6. **Рост модулями.** Новый домен = новые теги + новый дашборд. Не нужно менять архитектуру.

## Стек

| Компонент | Технология | Почему |
|-----------|-----------|--------|
| Desktop shell | **Tauri v2** | Лёгкий (30 МБ vs 150 МБ Electron), нативное окно macOS, file system access через Rust |
| Frontend | **React + TypeScript** | Большая экосистема, знакомый стек, много библиотек для drag-and-drop и визуализации |
| Drag-and-drop | **@dnd-kit** | Примитивы без навязанной структуры. Полный контроль над сеткой |
| Стилизация | **Tailwind CSS** | Быстрая разработка, консистентный дизайн |
| Валидация | **Zod** | Схемы данных в коде, валидация на лету, автовывод TypeScript-типов |
| State management | **Zustand** | Простой, без бойлерплейта, хорошо работает с файловым I/O |
| Backend (Tauri) | **Rust** | Минимальный — только чтение/запись файлов, file watcher, атомарные операции |
| AI-агент | **Claude Code / Cowork** | Внешний, работает с файлами напрямую |

## Фазы разработки

| Фаза | Название | Что получаем |
|------|---------|-------------|
| 1 | Foundation | Tauri + React проект, схемы данных (Zod), файловый I/O через Rust invoke, пустой shell с навигацией |
| 2 | Weekly Grid | Недельная сетка, блоки, создание/редактирование, навигация по неделям, баланс дня |
| 3 | Drag & Drop + Task Pool | @dnd-kit, перетаскивание блоков, resize, пул задач сбоку, drag из пула на сетку |
| 4 | Entities + Templates | CRUD сущностей, управление тегами, шаблон недели, переход между неделями |
| 5 | Dashboards | Рендерер JSX-компонентов, менеджер дашбордов, стартовый набор витрин |
| 6 | AI Integration | Очередь команд, file watcher, протокол агента, scheduling preferences |

Каждая фаза — самостоятельный рабочий результат, который можно запустить и пощупать.

## Файловая структура данных

```
tuzov-os/
├── data/
│   ├── schedule/
│   │   ├── 2026-w16.json
│   │   └── 2026-w17.json
│   ├── entities.json          # все сущности в одном файле
│   ├── templates/
│   │   └── default.json       # шаблон типичной недели
│   ├── config.json            # категории, цвета, preferences
│   └── dashboards/
│       ├── kpi.jsx
│       ├── monthly-plans.jsx
│       └── work-pipeline.jsx
├── commands/
│   ├── pending/               # агент кладёт команды сюда
│   ├── done/                  # выполненные
│   └── failed/                # с ошибками
├── src/                       # исходный код приложения
├── src-tauri/                 # Rust-бэкенд
└── spec/                      # эта спека
```
