## Архитектурные решения

| Вопрос | Решение |
|---|---|
| Overlay → Spotlight-like якорь | `top: 22vh`, max-width 640px, не модалка по центру. Это меньше визуального веса. |
| Backdrop blur | `backdrop-filter: blur(18px) saturate(140%)`. Saturate важен для macOS vibrancy. Если WebView не поддерживает — fallback `rgba(0,0,0,.55)` без blur. |
| Кнопка `↵` — passive kbd-chip | Не кликабелен. Submit — только Enter. Это паттерн Raycast/Linear/Spotlight. |
| Hint `⌘N · Тип: Задача` | Удаляем из overlay. В overlay юзер уже там — подсказка избыточна. Перенос в tooltip кнопки `+` («Создать ⌘N»). |
| Helper-line vs tooltip | Helper-line под input'ом для inline-modifiers подсказок (другой контент, не дубликат). Tooltip — про **как открыть** overlay. Не конфликтуют. |
| Триггер модификаторов | Оставляем `!` (явный prefix, не magicless NLP). Без триггера ломается на «завтра договориться» как title. `@` зарезервирован под area (фаза 3+), `#` под priority. |
| Парсер: токенизация vs match | Переходим на **токенизацию при каждом keystroke**. Возвращает `Token[]` с типом (`text` / `date-modifier`) и span'ами `{ start, end }`. Это enables подсветку, preview и click-to-toggle. |
| Подсветка фрагмента в input'е | **Overlay-слой паттерна `react-highlight-words`**: абсолютно позиционированный `<div>` под input'ом, ширина каждой буквы синхронизирована monospace-trick'ом или измерением. НЕ contenteditable — слишком сложно для одного модификатора. |
| Preview-строка | Под input'ом. Высота 18px, mono 12px gold для valid date, red для invalid. При отсутствии модификатора — `display: none` (не дёргать высоту). |
| Autocomplete popover | Появляется при вводе `!` в whitespace-границе или начале строки. Закрывается на Esc, click outside, Backspace удалившем `!`, ввод символа не матчащего ни одному пункту. Паттерн Slack/Bear. |
| Date picker | **Inline**, не модалка. Открывается из popover'а на месте popover'а (заменяет содержимое), тот же width. Календарь месячной сетки. |
| Click на подсвеченный токен | Toggle: одно состояние «parser активен» (default), другое «отключён, фрагмент — литерал». Хранится в Set<spanKey> в state QuickAdd. |
| Прошедшая дата (`!05.15` сегодня 31 мая) | Парсер возвращает дату текущего года (`2026-05-15`), preview подкрашивается оранжевым: «Прошло — 15 мая». **Не сдвигаем автоматически в +1 год**. Юзер сам решает. |
| Локализация | RU + латинские aliases (`!tom`, `!tomorrow`) как silent muscle-memory backup. Английский как primary — отложен. |
| Дни недели | Aliases `!пн`, `!вт`, `!ср`, `!чт`, `!пт`, `!сб`, `!вс` — ближайший такой день в будущем. |
| Будущие модификаторы (`@area`, `#priority`, `~30m`, `^parent`) | Архитектура должна подразумевать расширяемость: `getCompletionsForTrigger(trigger, query)` — универсальная функция, разные источники. **В этой фазе реализуем только `!`-trigger**, остальное — фазы 3+. |
| Parser — single-modifier policy MVP | Один `!`-модификатор за раз. Если юзер ввёл два (`!завтра !послезавтра`) — берём **последний** (это его новое намерение). Первый подсвечен серым с strikethrough как inactive. При submit вырезаются оба. |

### Шаг 1. Tokenizer + тесты (изолировано)

`src/services/quick-add-tokenizer.ts`:

```ts
export type TokenType =
  | "text"
  | "date-modifier"      // распознанная и валидная дата
  | "date-modifier-invalid"  // !13.40, !99.99 и т.п.
  | "date-modifier-past";    // прошедшая (для warning'а)

export interface Token {
  type: TokenType;
  start: number;       // индекс начала в input
  end: number;         // индекс после последнего символа
  raw: string;         // подстрока input.slice(start, end)
  // Только для date-modifier* :
  deadline?: string;   // ISO YYYY-MM-DD
  humanLabel?: string; // «завтра, чт 30 апр»
}

export function tokenize(input: string, baseDate?: Date): Token[];
```

Логика:

- Сканируем input посимвольно, ищем совпадения с регекспами по
  убыванию специфичности:
  1. `!\d{4}-\d{2}-\d{2}` — ISO.
  2. `!\d{2}\.\d{2}` — MM.DD текущего года.
  3. `!послезавтра` (раньше `!завтра` чтобы не false-positive).
  4. `!завтра`.
  5. `!пн|!вт|!ср|!чт|!пт|!сб|!вс` — ближайший такой день недели.
  6. `!пс` (= послезавтра), `!зв` (= завтра) — короткие aliases.
  7. `!tomorrow`, `!tom`, `!tmrw` — латинские aliases для завтра.
  8. `!через неделю` (одна неделя), `!через месяц`.
- Каждое совпадение — token, остальное — text-token'ы между ними.
- Все совпадения требуют **whitespace-границы**: либо start of
  input, либо `\s` слева; и либо end of input, либо `\s` справа.
- При нескольких modifier-токенах — все возвращаются. Решение
  «какой активный» — на уровне UI-компонента (берём последний).
- Для прошедшей даты (`!05.15` когда сегодня 31 мая того же года)
  возвращаем `type: "date-modifier-past"`, deadline — всё равно
  ISO текущего года.
- Для невалидной (`!13.40`, `!99.99`) — `date-modifier-invalid`,
  deadline — `null`.

Тесты (`quick-add-tokenizer.test.ts`):

1. `"Купить молоко"` → 1 text-token.
2. `"Купить молоко !завтра"` с baseDate=2026-04-29 → text-token + date-modifier (deadline=2026-04-30, humanLabel="завтра, чт 30 апр").
3. `"!послезавтра задача"` → date-modifier (5-1) + text-token.
4. `"!послезавтра"` сам по себе → date-modifier (deadline base+2) — НЕ false-positive `!завтра`.
5. `"Тест !05.15"` с baseDate=2026-04-29 → text + date-modifier.
6. `"Тест !05.15"` с baseDate=2026-06-01 → text + **date-modifier-past** (15 мая прошло).
7. `"Тест !13.40"` → text + **date-modifier-invalid**.
8. `"Тест !2026-12-31"` → text + date-modifier.
9. `"!завтра !послезавтра"` → 2 date-modifier'а (оба валидны).
10. `"!пн"` (понедельник) с baseDate в среду → date-modifier на ближайший пн.
11. `"!пн"` в понедельник → date-modifier на следующий пн (через 7 дней) — **не сегодня**.
12. `"!через неделю"` → date-modifier base+7.
13. `"!через месяц"` → date-modifier base+30 (упрощение).
14. `"!tomorrow"` → date-modifier base+1.
15. `"завтра договориться"` (без `!`) → 1 text-token. Не magic.
16. `"привет!завтра"` (без whitespace перед `!`) → 1 text-token.
17. `"!завтра!"` (без whitespace справа) → text-token «!завтра!» (граница нарушена).
18. `"  пробелы  !завтра  "` → token'ы корректно отражают span'ы (с учётом пробелов).
19. `"!завтра"` сам по себе → 1 date-modifier, span от 0 до 7.
20. Edge: пустая строка → пустой массив.

### Шаг 2. `formatDateRU` хелпер

`src/services/date-format-ru.ts`:

```ts
const DAYS_RU = ["вс", "пн", "вт", "ср", "чт", "пт", "сб"];
const MONTHS_RU = ["янв", "фев", "мар", "апр", "май", "июн",
                   "июл", "авг", "сен", "окт", "ноя", "дек"];

// "чт, 30 апр" — короткий human-readable формат для preview
// и popover'а.
export function formatDateRU(d: Date): string {
  const dow = DAYS_RU[d.getDay()];
  const day = d.getDate();
  const month = MONTHS_RU[d.getMonth()];
  return `${dow}, ${day} ${month}`;
}

// "завтра, чт 30 апр" — добавляет relative-prefix если уместно.
export function formatRelativeRU(d: Date, baseDate: Date): string {
  const diff = Math.round((d.getTime() - baseDate.getTime()) / (24 * 60 * 60 * 1000));
  const long = formatDateRU(d);
  if (diff === 0) return `сегодня, ${long}`;
  if (diff === 1) return `завтра, ${long}`;
  if (diff === 2) return `послезавтра, ${long}`;
  if (diff === -1) return `вчера, ${long}`;
  if (diff > 2 && diff < 8) return `через ${diff} дн., ${long}`;
  return long;
}
```

Используется и в tokenizer'е (для humanLabel), и в preview, и в
popover'е (вторичный текст справа от пункта).

### Шаг 3. ui-store расширение

```ts
export interface QuickAddState {
  open: boolean;
  type: QAType;
  category: string | null;
  lastCategory: string | null;
  // hintLabel: string;  // УДАЛИТЬ — больше не используется

```

Actions:
- `setQuickAddInput(input: string)` — обновляет input + пересчитывает tokens.
- `toggleTokenActive(span: string)` — добавляет/убирает span из deactivatedSpans.
- `openPopover(filter?: string)` / `closePopover()`.
- `setPopoverSelectedIndex(i: number)`.
- `applyPopoverItem(item: PopoverItem)` — вставляет в input полное название модификатора, закрывает popover.
- `openPicker()` / `closePicker()`.
- `setPickerSelectedDate(iso: string | null)`.
- `applyPickerDate()` — превращает выбранную дату в `!YYYY-MM-DD` в input'е, закрывает picker.

Старый `hintLabel` поле — удалить (его больше нет в UI).

`Set<string>` в Zustand state — Zustand сравнивает по reference, не по value, так что нужно создавать новый Set при каждом изменении: `new Set([...prev, span])`.

### Шаг 6. `QuickAddInput` с overlay-подсветкой

Самый нетривиальный кусок. Идея — у нас есть `<input>` для ввода
и абсолютно позиционированный `<div>` под/над ним с тем же
содержимым, но с подсветкой span'ов.

```tsx
// src/components/quick-add/QuickAddInput.tsx
export function QuickAddInput({ value, onChange, tokens, deactivatedSpans, ... }) {
  const inputRef = useRef<HTMLInputElement>(null);
  const overlayRef = useRef<HTMLDivElement>(null);

  // Sync scroll between input and overlay (если input scrollable)
  // — для длинного текста.

  return (
    <div className="qa-input-wrap">
      <div className="qa-input-overlay" ref={overlayRef} aria-hidden>
        {tokens.map((t) => {
          const span = `${t.start}-${t.end}`;
          const dezactivated = deactivatedSpans.has(span);
          if (t.type === "text") {
            return <span key={span}>{t.raw}</span>;
          }
          const cls = dezactivated
            ? "qa-token-highlight deactivated"
            : t.type === "date-modifier-invalid"
              ? "qa-token-highlight invalid"
              : t.type === "date-modifier-past"
                ? "qa-token-highlight past"
                : "qa-token-highlight";
          return (
            <span key={span} className={cls}
                  onClick={() => /* dispatch toggleTokenActive(span) */}>
              {t.raw}
            </span>
          );
        })}
      </div>
      <input
        ref={inputRef}
        className="qa-input"
        value={value}
        onChange={(e) => onChange(e.target.value)}
        onKeyDown={...}
      />
    </div>
  );
}
```

CSS:
```css
.qa-input-wrap { position: relative; height: 56px; padding: 0 18px; }
.qa-input-overlay {
  position: absolute; inset: 0; padding: 0 18px;
  font: 400 var(--fs-xl) var(--font);
  line-height: 56px;
  pointer-events: none;
  white-space: pre;
  color: transparent;  /* текст невидим, видна только подсветка */
}
.qa-input-overlay span { pointer-events: auto; }  /* для click-to-toggle */
.qa-input {
  position: relative; z-index: 1;
  width: 100%; height: 56px;
  background: transparent; border: 0;
  font: 400 var(--fs-xl) var(--font);
  color: var(--text-primary);
  caret-color: var(--accent);
  outline: none;
}
```

**Ключевой момент:** input'овский текст — primary, поверх (z=1).
Overlay под ним (z=0), показывает только подсветку (`color:
transparent`, фон рендерится, текст невидим но занимает место).
Идеальное выравнивание ширины букв требует **точно того же
шрифта** в обоих элементах (включая weight, size, letter-spacing,
line-height). Если шрифт пропорциональный (Outfit) — это
работает, потому что overlay рендерит ровно те же литералы.

Альтернатива (если выравнивание уйдёт): сделать input'овский
шрифт mono (`var(--mono)`) — тогда абсолютная синхронизация
ширины. Но визуально mono чуть менее «нативно» для ввода. Решать
по тестам — если выравнивание идеальное, оставлять Outfit.

### Шаг 8. Popover на `!`

`src/components/quick-add/QuickAddPopover.tsx`:

```tsx
interface PopoverItem {
  key: string;          // уникальный, для React key
  label: string;        // «завтра»
  secondary: string;    // «Чт, 30 апр»
  apply: string;        // что вставить в input'е («!завтра»)
  isAction?: "open-picker";  // для пункта «Выбрать дату…»
}

export function QuickAddPopover({ baseDate, filter, selectedIndex, onSelect, onClose }) {
  const items = useMemo(() => buildPopoverItems(baseDate, filter), [baseDate, filter]);
  // ...
}

function buildPopoverItems(baseDate: Date, filter: string): PopoverItem[] {
  const all: PopoverItem[] = [];
  // 1. Relative shortcuts
  all.push({ key: "tomorrow", label: "завтра",
             secondary: formatDateRU(addDays(baseDate, 1)),
             apply: "!завтра" });
  all.push({ key: "day-after", label: "послезавтра",
             secondary: formatDateRU(addDays(baseDate, 2)),
             apply: "!послезавтра" });
  all.push({ key: "next-week", label: "через неделю",
             secondary: formatDateRU(addDays(baseDate, 7)),
             apply: "!через неделю" });
  all.push({ key: "next-month", label: "через месяц",
             secondary: formatDateRU(addDays(baseDate, 30)),
             apply: "!через месяц" });
  // 2. Days of week (ближайшие 5)
  for (let d = 1; d <= 7; d++) {
    const date = addDays(baseDate, d);
    const dow = DAYS_RU_SHORT[date.getDay()];  // пн/вт/...
    all.push({ key: `dow-${dow}`, label: dow,
               secondary: formatDateRU(date), apply: `!${dow}` });
  }
  // 3. Action — выбрать дату...
  all.push({ key: "pick", label: "Выбрать дату…",
             secondary: "", apply: "", isAction: "open-picker" });
  
  // Filter
  const f = filter.toLowerCase();
  if (!f) return all;
  return all.filter((it) =>
    it.label.toLowerCase().startsWith(f) ||
    it.label.toLowerCase().includes(f)
  );
}
```

Положение: абсолютно под input'ом, ширина = ширине .qa-bar минус
36px паддинга (до краёв футтера). Z-index 305.

Hint-строка под списком (всегда видна, не фильтруется):
«`!2026-12-31` ISO · `!05.15` MM.DD».

Закрытие:
- Esc → `closePopover()`.
- Click outside → `closePopover()`.
- Backspace удалил `!` → `closePopover()`.
- Filter не матчит ни один пункт → автозакрытие через 200ms.

Применение пункта:
- `apply` (для обычных) → дописывает/заменяет фрагмент после `!`
  на полное `apply` значение, popover закрывается, parser
  пересчитывает tokens.
- `isAction === "open-picker"` → `openPicker()`, popover закрывается.

Навигация:
- ↑↓ — `setPopoverSelectedIndex` циклически.
- Enter / Tab — `applyPopoverItem(items[selected])`.
- При фильтрации — selectedIndex резетится в 0.

  но не обязательно. Если делать — `aria-live="polite"` для
  «Добавлено: <title>».
- Recent titles autocomplete (история ввода) — overkill v1.
- Live preview карточки в правой панели — splits attention,
  ломает focus model.
- Полноценный contenteditable input с atomic-delete токенов —
  слишком сложно для одного модификатора. Lite-overlay-вариант
  достаточен.
- Mobile / touch fallback — приложение pure desktop, отложено.
- Подсказка `?`-кнопка с полным cheatsheet'ом — onboarding-
  территория, отложено.

## Ловушки

- **Overlay-подсветка ≠ выравнивание.** Если шрифт пропорциональный
  (Outfit), браузер рендерит ширину каждой буквы детерминированно,
  но малейшая разница в font-weight / letter-spacing между
  input'ом и overlay'ем сломает выравнивание подсветки. Проверить
  на разных длинах текста, разных букв (особенно русские «ы», «ё»,
  «й»). Если ползёт — переключиться на mono для input'а.
- **Popover закрытие на Backspace.** Когда юзер удаляет `!`,
  оставшиеся символы (если они после `!` были) — попадают в text-
  token. Должны попасть, не должны странно подсвечиваться. Тестировать.
- **Tab inside popover.** Tab в popover'е применяет item, но
  обычно в overlay Tab — это переход на следующий элемент.
  В нашем случае — не нужно (focus всегда в input'е). `e.preventDefault()`
  обязателен.
- **Tab inside overlay (без popover'а).** Не должен ничего делать
  (мы не циклим фокус). preventDefault на Tab в KeyDown handler.
- **`Set<string>` в Zustand.** При мутации создавать новый Set,
  иначе useUIStore не отрендерит подписавшихся.
- **Tokenize при каждом keystroke.** Это OK по перформансу для
  640px input'а (~80 символов max в реалии), но если кто-то
  вставит в input огромный текст — может тормозить. Положить
  upper bound max-length=200 на input'е — safe-by-default.
- **Click-outside picker'а.** Picker внутри popover'а — клик по
  overlay (за пределами `.qa-bar`) закрывает overlay через
  существующий handler. Но клик по `.qa-bar` за пределами picker'а
  — не должен закрывать picker. Проверять `e.target` через
  `barRef.current.contains` и через `pickerRef.current.contains`.
- **`prefers-reduced-motion`** — обязательно отключить shake и
  qaIn анимации. Юзер может не любить движения по медицинским
  причинам.
- **Tooltip flicker.** Если tooltip имеет open-delay 600ms и
  hover-out-delay 0ms — быстрый mouseover/out не должен оставлять
  «висящий» tooltip. Реализация: `setTimeout` на open, `clearTimeout`
  на mouseLeave.
- **Tokenizer и weekend.** `!пн` в воскресенье — это завтра (1
  день), `!вс` в воскресенье — следующее воскресенье (7 дней).
  Тестировать оба случая.
- **`!завтра` в названии literally.** Юзер может захотеть написать
  «не забудь завтра» в title. Сейчас — нельзя без deactivation.
  Решение: либо escape (`\!завтра`), либо click-to-toggle (есть в
  фазе). Click-to-toggle — primary решение, escape — отложить.

## Что записать в memory

Принципиальные решения, которые применимы в фазах 3+:

- **Триггер-popover механизм универсальный**: `getCompletionsForTrigger(trigger,
  query)` — функция, разные trigger'ы (`!`, `@`, `#`, `~`, `^`)
  переиспользуют один UI. Phase 2.5 закладывает механику, фазы 3+
  расширяют список trigger'ов.
- **Парсер — токенизатор, не matcher.** Tokenize при каждом
  keystroke. Match-при-submit оставлен только для legacy
  `parseQuickAdd(input)` wrapper'а.
- **Inline-подсветка через overlay-слой**, не contenteditable.
  Предпочитаемый вариант для одного-двух модификаторов.
- **Past-date НЕ shift'им автоматически в +1 год** — показываем
  warning, пусть юзер решит. Predict ability важнее автокоррекции.
