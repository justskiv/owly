## Workflow внутри фазы

1. **Прочитай фазу целиком**. Не начинай делать первый пункт,
   пока не понял всю картину — пункты внутри группы часто
   зависят друг от друга по контексту.
2. **Re-grep `path:line` перед правкой**. Файл/строка могла
   уехать после прошлых коммитов или предыдущих правок R1.
3. **Делай группой**. Acceptance criteria и шаги реализации
   фазы разбиты на группы (A/B/C/D). Делай группу целиком,
   потом `task check`, потом дождись smoke от пользователя,
   потом atomic commit, потом следующая группа.
4. **Не bundle'ь группы**. Одна группа = один коммит. Это
   важно для smoke-теста — пользователь должен иметь
   возможность откатить логически связанный набор без
   разбора монокоммита.
5. **Не делай ничего из «Что НЕ включает»** — это в других
   фазах. Если нашёл ещё проблему рядом — добавь в
   `post-review-backlog.md`, не растягивай scope.
6. **Если мейнтанер просит править path:line, который не
   соответствует тому, что ты видишь** — это drift. Найди
   нужную сущность по символу/семантике, обнови файл фазы с
   новой строкой, потом правь.

## Пользовательский workflow (для агента, который ждёт смоук)

После каждой группы — пользователь делает smoke. Дождись его
явного «ок» прежде чем коммитить. Если он попросил
поправить — поправь, снова `task check`, снова smoke. Не
делай commit пока пользователь не сказал.

Если пользователь даёт smoke list в фазе, и ты не можешь его
выполнить сам (например, «drag блок 30+ секунд — нет лагов»),
это smoke ДЛЯ ПОЛЬЗОВАТЕЛЯ. Просто перечисли ему пункты — он
проверит руками и скажет, прошло ли.

## Что НЕ делать

- Не запускай `task dev` / `npm run tauri dev` сам — пользователь
  это делает (правило из `CLAUDE.md`).
- Не коммить без smoke-апрува пользователя на UI-меняющих
  правках.
- Не выходи за scope фазы. Если по дороге заметил smell —
  в `post-review-backlog.md`.
- Не правь `data/seed-v2/*` — это часть спеки.
- Не трогай `EntitiesPage` / `DashboardsPage` — kept per Phase 9
  D1/D2 как debug-входы (`Cmd+Shift+E`/`Cmd+Shift+D`).
- Не упоминай Claude/Anthropic в commit messages (правило из
  `~/.claude/CLAUDE.md`).
- Commit messages — английский, остальное (план/чат/доки) —
  русский.

## Если что-то пошло не так

- `task check` red после правки → разбираться, не bypass'ить
  hooks. Не использовать `--no-verify`.
- Path:line drift на первой же правке группы → обнови фазу
  (поправь номера строк), потом продолжай.
- Ловушки фаз (R1-R7 в каждой фазе) — это известные грабли.
  Если упёрся — перечитай ловушки.
