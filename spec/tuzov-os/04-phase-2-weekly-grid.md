> **Цель:** реализовать недельную сетку — ядро приложения. Блоки расписания, создание, редактирование, визуальный баланс дня.

## Компоненты

### WeekGrid.tsx (основной)

Сетка 7 колонок × временные строки. Вертикальная ось — время (с 06:00 до 23:00 видимая область, скролл до 00:00-05:00). Горизонтальная — дни недели.

│     │  14    │  15    │  16    │  17    │  18    │  19    │  20    │
├─────┼────────┼────────┼────────┼────────┼────────┼────────┼────────┤
│06:00│        │        │        │        │        │        │        │
├─────┤ Собаки │        │ Собаки │        │ Собаки │        │        │
│06:30│ (hlth) │        │ (hlth) │        │ (hlth) │        │        │
├─────┼────────┤        ├────────┤        ├────────┤        │        │
│07:00│ Завтрак│        │ Завтр. │        │        │        │        │
├─────┼────────┤        ├────────┤        │        │        │        │

**Реализация:**

- Каждая строка = 30 минут, высота строки = 40px (настраиваемо)
- Блоки позиционируются абсолютно внутри колонки дня
- Позиция top = (час * 60 + минуты) / 30 * ROW_HEIGHT
- Высота блока = duration / 30 * ROW_HEIGHT
- Цвет блока = цвет области из config (по полю `category` блока)
- Текущий час подсвечен горизонтальной линией (красной/акцентной)

### DayColumn.tsx

Одна колонка-день. Получает `date` и массив `blocks` для этого дня. Рендерит фон (сетка по 30 мин) и блоки поверх.

### TimeBlock.tsx

Один блок в расписании.

**Отображение:**
- Заливка цветом области (полупрозрачная)
- Левая граница цветом области (яркая, 3px)
- Текст: title (bold), время (light)
- Если блок < 30 мин — однострочный compact вид
- Если блок ≥ 60 мин — показать ещё notes (если есть)
- Статус: done = ✓ и приглушённый цвет, skipped = перечёркнутый

**Интерактивность:**
- Клик → открыть BlockEditor (modal)
- Правый клик → контекстное меню: Edit, Done, Skip, Delete

### BlockEditor.tsx

Модальное окно для создания/редактирования блока.

**Поля формы:**
- Title (text input, обязательное)
- Date (date picker или dropdown дней текущей недели)
- Start time (time picker, шаг 15 мин)
- Duration (number input или пресеты: 30, 60, 90, 120, 180 мин)
- Category (dropdown из config.areas — с цветом)
- Source entity (autocomplete поиск по entities — опционально)
- Status (radio: planned / done / skipped)
- Notes (textarea)

**Создание нового блока:**
- Клик на пустое место в сетке → открыть BlockEditor с предзаполненным date + start (время клика)
- Duration по умолчанию = 60 мин

**Редактирование:**
- Клик на блок → открыть BlockEditor со всеми данными блока

### DayBalance.tsx

Под каждым днём — компактная сводка.

```
Work: 5h | People: 1.5h | Health: 1h
Free: 6.5h
```

- Считать из blocks текущего дня
- Группировка по `category`
- Цвета из config.areas
- Free = 24h - сумма всех блоков (или waking hours: 16h - сумма)
- Компактное отображение: цветные полоски пропорционально времени

### WeekSummary.tsx

Итоги всей недели (над сеткой или в header):

```
Неделя 16: Work 23h | Health 7h | Growth 3.5h | People 4h | Life 2h | Free 30.5h
```

Процентные полоски по категориям. Сравнение с прошлой неделей (если файл существует).

## Навигация по неделям

В Header:
- `←` / `→` — предыдущая / следующая неделя
- Кнопка "Сегодня" — перейти к текущей неделе
- Отображение: "Неделя 16 · 13–19 апреля 2026"

При переключении:
1. Сохранить текущую неделю (если были изменения)
2. Загрузить новую неделю из файла
3. Если файла нет — показать пустую неделю (с предложением создать или применить шаблон)

## Расчёт номера недели

ISO 8601: `getISOWeek()` из date-fns. Формат: `YYYY-wWW` (например `2026-w16`).

Установить:
```bash
npm install date-fns
```

## Overlap detection

Если два блока пересекаются по времени в одном дне — оба получают визуальную индикацию (красная рамка или предупреждающий значок). Приложение **не запрещает** overlap, только подсвечивает.

```typescript
function hasOverlap(blockA: Block, blockB: Block): boolean {
  if (blockA.date !== blockB.date) return false;
  const startA = timeToMinutes(blockA.start);
  const endA = startA + blockA.duration;
  const startB = timeToMinutes(blockB.start);
  const endB = startB + blockB.duration;
  return startA < endB && startB < endA;
}
```

## Стилизация блоков

Каждая область (area) имеет свой цвет из `config.json`. Блок окрашивается по полю `category`:

```
work     → #3B82F6 (синий)
people   → #EC4899 (розовый)
life     → #F59E0B (жёлтый)
growth   → #8B5CF6 (фиолетовый)
health   → #10B981 (зелёный)
```

Блок: фон = цвет с opacity 15%, левая граница = цвет 100%.
