# Фаза 3: Drag & Drop + Task Pool

> **Цель:** добавить полноценный drag-and-drop в сетку и боковую панель с пулом задач.
>
> **Результат:** можно перетаскивать блоки между днями и временами, растягивать за край, перетаскивать задачи из пула на сетку. Это превращает сетку из статичной картинки в интерактивный инструмент планирования.
>
> **Предусловие:** Фаза 2 завершена, сетка рендерит блоки, создание/редактирование работает.

## Drag-and-drop блоков в сетке

### Перемещение блока

Пользователь хватает блок → перетаскивает на другой день/время → отпускает.

**Реализация:**

1. Каждый `TimeBlock` обёрнут в `useDraggable` из @dnd-kit
2. Каждая 30-мин ячейка в `DayColumn` — `useDroppable`
3. При drop: определить новый `date` и `start` по позиции droppable
4. Snap to 15-min grid: округлить `start` до ближайших 15 минут
5. Обновить блок в store → автосохранение

**Визуальная обратная связь:**
- При drag: блок становится полупрозрачным на исходном месте, "призрак" следует за курсором
- Над валидным drop target: подсветка ячейки
- Если drop создаст overlap: подсветка жёлтым (предупреждение, но не запрет)

### Изменение длительности (resize)

Пользователь тянет за нижний край блока → изменяет длительность.

**Реализация:**

1. В нижней части `TimeBlock` (последние 8px) — зона resize
2. При mousedown на зоне resize → начинаем tracking mousemove
3. Вычисляем новую длительность по позиции курсора: `newDuration = Math.round(deltaY / ROW_HEIGHT) * 30`
4. Минимум: 15 минут
5. Snap to 15-min increments
6. При mouseup → обновить `duration` в store

**Визуальная обратная связь:**
- Курсор `ns-resize` при наведении на зону resize
- При resize: блок визуально растягивается/сжимается в реальном времени
- Показать tooltip с новой длительностью: "1h 30m"

### Клик vs drag

Проблема: клик на блок должен открывать редактор, но drag тоже начинается с mousedown.

**Решение:** если между mousedown и mouseup прошло < 200ms и курсор сдвинулся < 5px → это клик (открыть редактор). Иначе — это drag. @dnd-kit обрабатывает это через `activationConstraint`:

```typescript
useSensor(PointerSensor, {
  activationConstraint: {
    distance: 5,  // минимум 5px движения для начала drag
  }
})
```

## Пул задач (TaskPool)

Боковая панель справа от сетки. Показывает незапланированные сущности из всех областей.

### Что попадает в пул

Сущность попадает в пул если:
1. `type` ∈ ["task", "project", "event", "routine"] (не note, не metric, не contact, не goal)
2. `status` === "active" (не done, не archived, не someday)
3. Нет блока в текущей неделе с `source_entity_id` равным ID сущности

Логика `getUnscheduled(weekBlocks)` в entity store.

> **Примечание:** в Фазе 3 пул работает на данных из entities.json (загруженных в store в Фазе 1). Полноценное UI для создания/редактирования сущностей появится в Фазе 4. В Фазе 3 новые сущности можно добавить через seed-скрипт или ручное редактирование entities.json.

### TaskPoolItem.tsx

Карточка задачи в пуле:
- Иконка типа (📝 task, 📁 project, 📅 event, 🔄 routine)
- Title
- Estimated time (если есть): "2h", "30m"
- Теги (цветные точки)
- Deadline (если есть и близок): "до 18 апр"
- Для контактов: "overdue 3d" если `last_contact` + `desired_cadence_days` < today

### Drag из пула на сетку

1. Каждый `TaskPoolItem` обёрнут в `useDraggable`
2. При drop на ячейку сетки → создать новый блок:

```typescript
{
  id: generateId("blk"),
  title: entity.title,
  date: dropTarget.date,
  start: dropTarget.time,
  duration: entity.estimated_minutes || 60,  // fallback 60 мин
  category: entity.tags[0] || "work",        // первый тег как категория
  source_entity_id: entity.id,
  status: "planned",
  notes: ""
}
```

3. Задача исчезает из пула (т.к. теперь есть блок с её ID)
4. Если задачу удалить из сетки → она снова появится в пуле

## DnD Context

Общий `DndContext` оборачивает и сетку и пул. Это позволяет drag из пула на сетку.

```tsx
// PlannerPage.tsx
<DndContext
  sensors={sensors}
  onDragStart={handleDragStart}
  onDragEnd={handleDragEnd}
>
  <div className="flex flex-1">
    <WeekGrid />
    <TaskPool />
  </div>
  <DragOverlay>
    {activeDragItem && <TimeBlockPreview block={activeDragItem} />}
  </DragOverlay>
</DndContext>
```

## Keyboard shortcuts

- `Delete` / `Backspace` на выбранном блоке → удалить
- `D` на выбранном блоке → mark as done
- `S` на выбранном блоке → mark as skipped
- `N` → создать новый блок (открыть BlockEditor)
- `T` → toggle Task Pool
- `←` / `→` → предыдущая / следующая неделя

Реализовать через `useEffect` + `addEventListener('keydown', ...)` на уровне PlannerPage.
