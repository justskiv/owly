# Фаза 4: Entities + Templates + Week Lifecycle

> **Цель:** полноценное управление сущностями, шаблон рутинной недели, переход между неделями.
>
> **Результат:** можно создавать/редактировать/удалять сущности любого типа. При создании новой недели — автозаполнение из шаблона. Незавершённые задачи предыдущей недели видны в пуле.
>
> **Предусловие:** Фазы 1-3 завершены.

## Контекст

Прочитай `01-data-schema.md` — там описаны все типы сущностей с полями. Прочитай `02-architecture.md` — структура компонентов.

### EntityList.tsx

Список карточек. Каждая карточка показывает:
- Иконка типа
- Title
- Теги (цветные badges)
- Тип-специфичная информация:
  - task: deadline, priority, parent project
  - project: pipeline_stage, количество задач
  - routine: frequency, days
  - event: date, time, location
  - contact: cadence, last_contact, overdue?
  - goal: progress (current / target)
  - note: первые 100 символов body
  - metric: current_value + unit, trend (↑↓→)

Клик на карточку → открыть EntityEditor.

### EntityEditor.tsx

Модальное окно или right panel для создания/редактирования.

**Общие поля (все типы):**
- Title (text, обязательное)
- Type (select, при создании; при редактировании — read-only)
- Tags (multi-select с цветами, можно добавить кастомный)
- Status (select: active, someday, done, archived)
- Priority (select: high, medium, low, нет)
- Deadline (date picker, опционально)
- Estimated time (number input + select min/hours, опционально)

**Тип-специфичные поля:**
Форма динамически показывает дополнительные поля в зависимости от `type`. См. `01-data-schema.md` для полного списка полей каждого типа.

**Примеры:**
- `type === "task"` → показать: parent project (autocomplete по проектам), checklist
- `type === "project"` → показать: description (textarea), pipeline stage (select), child tasks (список с + для добавления)
- `type === "contact"` → показать: desired cadence, last contact (date), travel time, important dates (repeatable), notes

**Кнопки:**
- Save — сохранить (создать или обновить)
- Delete — удалить (с подтверждением)
- Cancel — закрыть без сохранения

### TagFilter.tsx

Переиспользуемый компонент фильтрации по тегам:

## Часть 3: Переход между неделями

### Создание новой недели

Когда пользователь переходит на неделю, для которой нет файла:

1. Показать диалог: "Неделя 17 (20-26 апреля) ещё не создана"
   - Кнопка "Создать из шаблона" — применить дефолтный шаблон
   - Кнопка "Создать пустую" — пустая неделя
2. При создании из шаблона:
   - Читать `data/templates/default.json`
   - Для каждого template block → создать Block с:
     - Пересчитанной датой (template.day → конкретная дата недели)
     - `status: "planned"`
     - `source_entity_id: null`
   - Сохранить как `data/schedule/2026-w17.json`

### WeekManager service

**src/services/week-manager.ts:**

```typescript
interface WeekManager {
  // Создать новую неделю
  createWeek(weekId: string, applyTemplate?: string): Promise<WeekFile>;

  // Получить незавершённые задачи прошлой недели
  getCarryOver(currentWeekId: string): Promise<Entity[]>;

  // Получить номер недели по дате
  getWeekId(date: Date): string;

  // Получить даты начала и конца недели
  getWeekRange(weekId: string): { start: Date; end: Date };

  // Список существующих недель (для истории)
  listWeeks(): Promise<string[]>;
}
```

### Carry-over (перенос незавершённых задач)

При открытии новой недели — показать секцию в пуле задач:

**"Незавершённые с прошлой недели"** — блоки из предыдущей недели со статусом `planned` (не done, не skipped), у которых есть `source_entity_id`.

Это не автоматический перенос блоков. Это просто сущности, которые были запланированы но не выполнены — они и так в пуле (потому что в новой неделе нет блоков для них). Нужно только визуально выделить их как "перенесённые".

**Реализация:**
- В `TaskPool` добавить секцию "С прошлой недели" (collapsed по умолчанию, с badge-счётчиком)
- Определять: загрузить предыдущую неделю, найти блоки `status === "planned"` с `source_entity_id`, вернуть эти entity IDs
- В пуле: эти сущности показываются в отдельной секции сверху

## Часть 4: Настройки

### SettingsPage.tsx (или модал)

Доступ: иконка шестерёнки в sidebar (внизу).

**Секции:**
- **Области жизни** — CRUD по areas: название, цвет, иконка. Можно добавлять свои.
- **Шаблон недели** — ссылка на TemplateEditor
- **Пайплайн** — редактирование стадий pipeline для проектов
- **Данные** — путь к data/, кнопка "Открыть папку в Finder"

> **Примечание:** секция "AI-планирование" (scheduling preferences) добавляется в Settings в Фазе 6, когда реализуется очередь команд и AI-интеграция.
