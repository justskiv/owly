follow Semantic Versioning. Curated by hand — only things a user sees
land here. Internal refactors, tests, CI, build configs, and docs
for developers / agents go in commit messages, not in this file.

release, rename it to `## [X.Y.Z] — YYYY-MM-DD` and add a fresh empty
`## [Unreleased]` on top.
